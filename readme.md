It includes:\n\n- Modern **C++26 / C23** source for `rastertokpsl` (Kyocera raster filter)\n- Automated **CMake ≥3.25** build and install system with Ninja Multi‑Config generator\n- Bundled PPD files, filter binaries, and installation scripts\n- CMake workflow presets for **GCC** and **Clang** toolchains\n\n**Supported platform:** Linux x86_64 with CUPS (requires `libcups` / `cups-devel`).\n\n---\n\n## Important Notice — Kyocera Driver Model Transition\n\n**Kyocera Document Solutions Inc.** has shifted from per‑model manual PPD distribution to a **Universal Driver Model** and cloud‑centric print solutions such as [Universal Print](https://www.kyoceradocumentsolutions.com/support/universal_print/). As a direct result, the legacy automated PPD download endpoints are no longer maintained by the manufacturer.\n\n| Evidence | Source |\n|---|---|\n| Kyocera officially catalogs products that natively support Universal Print | [KYOCERA — Models Supporting Universal Print](https://www.kyoceradocumentsolutions.com/support/universal_print/) |\n| The global download portal no longer exposes per‑model legacy PPD APIs | [KYOCERA Global Download & Support Portal](https://global.kyocera.com/support/download/) |\n| Current Linux driver packages are published as consolidated “Linux Universal Driver” releases | [KYOCERA EU — Linux Universal Driver (Phase 9.4)](https://www.kyoceradocumentsolutions.eu/en/support/downloads.name-L2V1L2VuL3ByaW50ZXJzL0VDT1NZU1AzMTU1RE4=.html) |\n| Legacy PPD collections are now preserved only by the OpenPrinting archive | [OpenPrinting — Kyocera PPD Archive](https://www.openprinting.org/download/PPD/Kyocera/en/) |\n\nBecause upstream no longer maintains the legacy download infrastructure, this project **disables automated PPD downloading by default** (`DOWNLOAD_PPDS=OFF`). The default build path uses the bundled `ppd/` directory. Users who require external drivers should obtain current universal packages directly from [KYOCERA Document Solutions](https://global.kyocera.com/support/download/).\n\n---\n\n## Architecture\n\n### CUPS Filter Pipeline\n\nThe diagram below shows how a print job flows through CUPS and reaches the Kyocera printer via this driver.\n\n```mermaid\nflowchart LR\n    A[Application<br>PDF / PS / Image] -->|stdin / file| B[CUPS Scheduler<br>cupsd]\n    B --> C[pdftops / imagetoraster<br>CUPS built-in filter]\n    C --> D[rastertokpsl<br>Kyocera KPSL filter]\n    D -->|KPSL byte stream| E[USB / TCP / LPD<br>Printer backend]\n    E --> F[Kyocera Printer<br>FS-1020MFP / FS-1040 / ...]\n```\n\n### Build System Architecture\n\n```mermaid\nflowchart TD\n    A[CMake ≥3.25<br>Ninja Multi-Config] --> B{Toolchain Preset}\n    B -->|gcc| C[GCC Compiler<br>C23 / C++26]\n    B -->|clang| D[Clang / LLVM<br>C23 / C++26]\n    C --> F[Open Source Driver<br>src/]\n    D --> F\n    F --> G[rastertokpsl binary]\n    G --> H[ppd/ bundled PPDs]\n    I[Proprietary Binaries<br>proprietary/] --> J[kyocera_ppd-config<br>DOWNLOAD_PPDS=OFF]\n    J --> H\n    K[INSTALL_ORIGINAL_PROPRIETARY_DRIVERS=ON] --> I\n```\n\n---\n\n## Open Source vs Proprietary Driver\n\n> **Recommendation:** By default the project installs the **proprietary bundled driver** (`INSTALL_ORIGINAL_PROPRIETARY_DRIVERS=ON`). The open-source reverse-engineered build is provided for transparency, research, and environments that cannot ship proprietary binaries, but it is **functionally inferior** to the manufacturer reference.\n\n| Feature | Proprietary Bundled | Open Source (`src/`) |\n|---|---|---|\n| Page orientation (portrait ↔ landscape) | **Full support** | Partial — may print incorrectly |\n| Paper size auto-detection | **Supported** | Limited / manual selection required |\n| All page formats (A4, A5, Letter, Legal, etc.) | **Supported** | Not all formats validated |\n| Halftone / dithering accuracy | Reference quality | Reverse-engineered approximation |\n| CUPS argument order | Corrected wrapper | Original order may misroute options |\n| License | Proprietary binary (shipped) | GPL-3.0 (rebuildable) |\n| Auditability / Reproducibility | Opaque | Full source available |\n\n### When to use the open-source driver\n\n- You are auditing, fuzzing, or contributing to a clean-room implementation.\n- Your distribution policy prohibits shipping proprietary blobs.\n- You have validated the output against your exact model and page setup.\n\n### When to use the proprietary bundled driver\n\n- **Default choice for daily printing.** It is the only configuration tested against all supported models and page formats.\n- You need reliable portrait / landscape switching.\n- You require maximum halftone fidelity on GDI-based Kyocera devices.\n\nTo explicitly opt into the open-source build:\n\n```bash\ncmake --preset gcc -DINSTALL_ORIGINAL_PROPRIETARY_DRIVERS=OFF\ncmake --build --preset gcc\nsudo cmake --install build/gcc --prefix /usr\n```\n\n---\n\n## Supported Models\n\nThis driver package provides bundled PPDs and filter support for the following Kyocera preset support)\n- `ninja` (Ninja Multi‑Config generator)\n- `g++` (C++26) and `gcc` (C23) **or** `clang++` / `clang`\n- `libstdc++` and `libstdc++-devel`\n- CUPS development headers\n\n```bash\n# Fedora\nsudo dnf install cups-devel cmake ninja-build gcc g++ libstdc++-devel\n\n# Ubuntu / Debian\nsudo apt install libcups2-dev cmake ninja-build gcc g++ libstdc++-12-dev\n```\n\n---\n\n## Build & Install\n\nThe project provides CMake presets for GCC and Clang. On Linux the supported toolchains are **GCC** and **Clang**.\n\n### 1. Clone\n\n```bash\ngit clone https://github.com/e-gleba/kyocera-drivers.git\ncd kyocera-drivers\n```\n\n### 2. Configure\n\n```bash\n# GCC (recommended for Linux)\ncmake --preset gcc\n\n# LLVM Clang\ncmake --preset clang\n```\n\n### 3. Build\n\n```bash\ncmake --build --preset gcc\n# or\ncmake --build --preset clang\n```\n\nYou may also build directly against the generated tree:\n\n```bash\ncmake --build build/gcc --parallel\n```\n\n### 4. Install (system‑wide)\n\n```bash\nsudo cmake --install build/gcc --prefix /usr\n```\n\n---\n\n## Installation Components\n\nCMake install components are provided for selective deployment:\n\n| Component | Command |\n|---|---|\n| Runtime only | `cmake --install build/gcc --component runtime` |\n| Development | `cmake --install build/gcc --component devel` |\n\n---\n\n## Uninstall\n\nCMake tracks installed files in `install_manifest.txt`. To remove them:\n\n```bash\ncd build/gcc\nsudo xargs rm -f < install_manifest.txt\n```\n\nIf the `uninstall` target is present in your build tree:\n\n```bash\nsudo cmake --build . --target uninstall\n```\n\n*(If the target is missing, see the [CMake Installing and Testing tutorial](https://cmake.org/cmake/help/latest/guide/tutorial/Installing%20and%20Testing.html) for the standard uninstall recipe.)*\n\n---\n\n## Usage\n\nAfter installation, CUPS recognizes the bundled Kyocera PPDs and filters. Add a printer via the CUPS web UI or `lpadmin`, selecting the installed Kyocera driver.\n\nTo exercise the filter directly:\n\n```bash\n./build/gcc/rastertokpsl <args>\n```\n\nRun `--help` for argument details.\n\n---\n\n## Troubleshooting\n\n| Symptom | Resolution |\n|---|---|\n| Installation fails with permission errors | Ensure `/usr/share/cups/model/Kyocera` and `/usr/lib/cups/filter` are writable (requires root). |\n| Missing dependencies during configure | Install `cups-devel` (Fedora) or `libcups2-dev` (Ubuntu) and verify CMake is on PATH. |\n| Verbose build logs needed | Append `--verbose` to the build command: `cmake --build . --verbose` |\n| Filter runtime errors | Inspect `/var/log/cups/error_log` for CUPS‑level diagnostics. |\n| Landscape prints as portrait (or vice versa) | You are using the open-source driver. Re-install with the proprietary bundled driver enabled (default). |\n| Page cuts off or wrong paper size detected | Ensure the PPD matches your model. If the issue persists, switch to the proprietary driver. |\n\n---\n\n## References & Evidence\n\n- [CMake Tutorial — Installing and Testing](https://cmake.org/cmake/help/latest/guide/tutorial/Installing%20and%20Testing.html)\n- [SDB: Using Your Own Filters to Print with CUPS](https://en.opensuse.org/SDB:Using_Your_Own_Filters_to_Print_with_CUPS)\n- [Ghidra — Software Reverse Engineering Framework](https://ghidra-sre.org/)\n- [Original Reverse‑Engineering Repository](https://github.com/sv99/rastertokpsl-re)\n- [KYOCERA Document Solutions — Models Supporting Universal Print](https://www.kyoceradocumentsolutions.com/support/universal_print/)\n- [KYOCERA Global Download & Support Portal](https://global.kyocera.com/support/download/)\n- [OpenPrinting — Kyocera Legacy PPD Archive](https://www.openprinting.org/download/PPD/Kyocera/en/)\n\n---\n\n## License\n\nThis project is licensed under the **GNU General Public License v3.0**.
